# دليل تثبيت وتشغيل تطبيق مصروف البيت

## المتطلبات
- Node.js (الإصدار 18 أو أحدث)
- npm أو pnpm

## خطوات التثبيت

### 1. استخراج الملفات
```bash
unzip expense-tracker-app.zip
cd expense-tracker
```

### 2. تثبيت المكتبات
```bash
pnpm install
# أو
npm install
```

### 3. إعداد قاعدة البيانات
```bash
pnpm db:push
# أو
npm run db:push
```

### 4. تشغيل التطبيق
```bash
pnpm dev
# أو
npm run dev
```

التطبيق سيعمل على: `http://localhost:3000`

## الميزات الرئيسية

### 1. صفحة البداية المحمية
- واجهة آمنة مع حماية كلمة السر
- حفظ الجلسة في localStorage

### 2. إدارة المعاملات
- إضافة دخل ومصروفات
- تعديل المعاملات المسجلة
- حذف المعاملات
- عرض قائمة شاملة للمعاملات

### 3. التصنيفات
**تصنيفات الدخل:**
- راتب
- دخل إضافي
- مشاريع جانبية
- مكافأة
- استثمارات
- أخرى

**تصنيفات المصروفات:**
- مصروفات ثابتة (SGK، إيجار، عائدات، فواتير)
- اشتراكات مدفوعة
- كورسات ودروس (الشيخ، إرساء)
- مصروفات سيارة (بنزين، مخالفات، تصليح، ضرائب)
- طعام
- سوبرماركت
- مواصلات

### 4. الإحصائيات
- رسوم بيانية تفاعلية
- توزيع المصروفات حسب الفئات
- إحصائيات شهرية

### 5. الميزانية
- تحديد ميزانية شهرية
- تنبيهات عند الاقتراب من حد الإنفاق
- عرض نسبة الإنفاق

### 6. التقارير الشهرية
- تقارير مفصلة لكل شهر
- مقارنة الشهور والسنوات
- تصفح حسب الشهر والسنة
- تحميل التقارير

### 7. الاستيراد والتصدير
- تصدير البيانات بصيغة JSON و CSV
- استيراد البيانات من ملفات سابقة
- دليل شامل لحفظ البيانات على Google Drive

## حقول المعاملة

كل معاملة تتضمن:
- **المبلغ**: المبلغ المالي
- **الفئة**: تصنيف المعاملة
- **التاريخ**: تاريخ المعاملة
- **الشخص المسؤول**: من قام بالصرف أو الإيداع
- **الوصف**: تفاصيل إضافية عن المعاملة

## نصائح الاستخدام

1. **النسخ الاحتياطية**: استخدم ميزة الاستيراد والتصدير لحفظ بياناتك بانتظام
2. **Google Drive**: احفظ ملفات التصدير على Google Drive للوصول إليها من أي جهاز
3. **الميزانية**: حدد ميزانية شهرية معقولة لتتبع نفقاتك بشكل أفضل
4. **التقارير**: راجع التقارير الشهرية لفهم أنماط الإنفاق

## استكشاف الأخطاء

### المشكلة: كلمة السر لا تعمل
**الحل:** تأكد من إدخال كلمة السر بشكل صحيح: `2599423`

### المشكلة: قاعدة البيانات لا تتصل
**الحل:** تأكد من تشغيل أمر `pnpm db:push` بنجاح

### المشكلة: البيانات لا تظهر
**الحل:** امسح ذاكرة التخزين المؤقت (Cache) وأعد تحميل الصفحة
